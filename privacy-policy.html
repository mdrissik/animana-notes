Animana Notes AI

Chrome Extension

Privacy Policy

Last updated: April 2026

Summary: Animana Notes AI stores your API key and templates locally on your device only. It reads patient data from the Animana page you have open. It sends consultation notes and patient data to Anthropic's API to generate text. No data is sent to any other server. No data is collected by the extension developer.

1. What data the extension accesses

Animana Notes AI reads the following information from the IDEXX Animana page you have open in your browser:

  • Patient name, species, breed, sex, age, and weight
  • Microchip number and passport number
  • Client name and CPH number
  • Clinical history entries visible on the patient history page

This data is read directly from the web page you are already viewing — the extension does not access any Animana API, database, or server independently.

2. What data is sent to third parties

When you click Generate or Summarise, the extension sends the following to the Anthropic API (anthropic.com):

  • The patient information scraped from the current Animana page
  • The consultation notes you typed into the extension
  • Patient history entries from the current page

This data is sent to Anthropic solely to generate the SOAP note text. Please refer to Anthropic's Privacy Policy for how they handle API request data. Anthropic does not use API request data to train models by default.

No data is sent to the extension developer or any other third party.

3. What data is stored locally

The extension stores the following in your browser's local storage (chrome.storage.local), which remains on your device only:

  • Your Anthropic API key
  • Custom consultation note templates you create
  • Your custom AI system prompt (if set)

This data never leaves your device except as described in Section 2 above (your API key is used to authenticate requests to Anthropic's API).

4. Patient data and clinical confidentiality

You are responsible for ensuring your use of this extension complies with applicable data protection laws (including GDPR where applicable) and your clinic's data handling policies. Patient data is sent to Anthropic's API when you generate notes — you should ensure this is consistent with your practice's agreements and obligations.

The extension developer is a sole user of this tool and does not receive, store, or process any patient data.

5. Permissions used

The extension requests the following Chrome permissions:

  • activeTab / scripting — to read patient data from the Animana page you have open and to fill the consult form fields
  • storage — to save your API key and templates locally on your device
  • tabs / webNavigation — to detect which Animana page is open and navigate to the consult form
  • Host permission for animana.com — to access the Animana pages you are logged into
  • Host permission for api.anthropic.com — to send requests to the Anthropic API for note generation

6. Data retention

The extension does not retain any data beyond your current browser session, except for the items stored locally (API key, templates, system prompt) which persist until you clear them or uninstall the extension.
